お名前.com Navi での操作手順

  1. お名前.com Navi にログイン
  2. 「ドメイン」 → 「DNS設定」
  3. 対象ドメイン hrtep.com の「DNS設定 / 転送設定」を選択
  4. 「DNSレコード設定を利用する」を選択して「次へ」
  5. 上記 4 件をそれぞれ入力
    • 1〜3（CNAME）: ホスト名欄に xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx._domainkey.send の部分のみ入力（末尾の .hrtep.com は自動付加）／ TYPE: CNAME ／ VALUE: 表の値
    • 4（TXT）: ホスト名欄に send のみ入力／ TYPE: TXT ／ VALUE: v=spf1 include:amazonses.com ~all
  6. 4 件入力後「追加」を押し、「確認画面へ進む」 → 内容確認の上「設定する」を押下
絶対に変更しないでください 以下のレコードに触れると受信メールやドメイン認証が壊れます。新規 4 件の追加のみが本作業の対象です。
  • hrtep.com の MX レコード（Google Workspace 等 6 件）
  • hrtep.com apex の SPF レコード（apex の TXT）
  • _dmarc.hrtep.com の DMARC レコード（TXT）
  • hrtep.com apex の A レコード（現行 WordPress IP 118.27.122.212） — Phase 3 で別途依頼予定
  • その他、本書面に列挙されていない既存のあらゆるレコード

SES サンドボックス継続運用について

AWS の本番化申請（Production Access）は取得しません。理由：

  • HRtep の問合せ件数想定（月 30〜100 件 = 1 日 1〜3 件）に対し、サンドボックスの上限（1 日 200 件）は 60 倍以上の余裕
  • 送信先は tsukahira@hrtep.com 1 アドレスに固定。サンドボックスの「Verified Identity 限定」制約は影響しない
  • 必要に応じて将来 Production Access に切替可能。サンドボックスから本番運用へは 1 日で移行できる

もしものときのロールバック

Phase 3 切替後に問題（HTTPS 表示不可・フォーム送信失敗・受信メール影響など）が発生した場合は、安井様にて A レコードを元の値（118.27.122.212）に戻すだけで旧サイトに復帰します。DNS の TTL は 1 時間設定なので、最長 1 時間で完全復旧。
特に 受信メール（MX レコード）に意図せず変更が起きた場合は最優先でロールバックとする運用です。
