05お問合せフォームの実装状態（reCAPTCHA + SES）

SES Production Access 申請文の commitment（同意必須・bot 排除）と実装が一致している状態を担保。

動作経路ユーザー入力 → zod バリデーション → reCAPTCHA v3 トークン取得 → サーバー側で Google siteverify → SES SendEmail（CONTACT_EMAIL_TO へ）

5.1 クライアント側（components/contact/ContactForm.tsx）

  • react-hook-form + zod（@hookform/resolvers/zod）でフロント検証
  • 同意チェックボックス（agreement: z.literal(true)）必須化、未チェックで送信不可
  • 送信時に grecaptcha.execute(SITE_KEY, { action: 'contact_submit' }) でトークン取得
  • 取得したトークンを Server Action へ POST
  • reCAPTCHA バッジ + Google プライバシー/利用規約リンク表示済（reCAPTCHA Terms 準拠）

5.2 サーバー側（app/contact/actions.ts）

  • Server Action で zod 再検証（クライアント側回避を防止）
  • reCAPTCHA トークンを Google siteverify エンドポイントに POST、score ≥ 0.5 で許可
  • 検証 OK の場合のみ SESv2Client.send(SendEmailCommand) を発火
  • 送信元: noreply@send.hrtep.com ／ Reply-To: フォーム送信者本人 ／ Configuration Set 経由
  • CONTACT_EMAIL_TO 未設定時は暫定無効化分岐（受付完了表示のみ・実メール送信なし）

5.3 ライブ動作確認（仮 URL 上で実証済）

E2E 検証完了 https://main.d3561ojkga3nyn.amplifyapp.com/contact でフォーム送信を実行 → (a) grecaptcha が長さ 2105 のトークンを返却 → (b) サーバー側で Google siteverify 通過（success=true / score ≥ 0.5）→ (c) CONTACT_EMAIL_TO 未設定分岐に到達し「お問い合わせを受け付けました」表示。 reCAPTCHA verify が成立していなければ「送信検証に失敗しました」が返るため、上記成功表示自体が verify path 動作の証跡。

8.2 お名前.com Navi での操作手順

  1. お名前.com Navi にログイン
  2. 「ドメイン」 → 「DNS設定」
  3. 対象ドメイン hrtep.com の「DNS設定 / 転送設定」を選択
  4. 「DNSレコード設定を利用する」を選択して「次へ」
  5. 入力欄に上記 4 件をそれぞれ入力
    • CNAME 3 件: ホスト名欄に xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx._domainkey.send の部分のみを入力（末尾の .hrtep.com は自動付加）／ TYPE: CNAME ／ VALUE: 上記の値（末尾は .dkim.amazonses.com）
    • TXT 1 件: ホスト名欄に send のみを入力（末尾の .hrtep.com は自動付加）／ TYPE: TXT ／ VALUE: v=spf1 include:amazonses.com ~all
    • TTL はすべて既定値（3600）でかまいません
  6. 4 件分入力したら「追加」を押し、「DNSレコード設定用テンプレート」のチェックを外したまま「確認画面へ進む」
  7. 内容確認の上「設定する」を押下
