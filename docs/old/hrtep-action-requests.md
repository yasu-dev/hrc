## ACTION 02 — DMARC レポート受信先アドレスの現存確認

### 背景

`hrtep.com` の DMARC レコードに、ドメイン認証の運用レポート（aggregate report）の受信先として以下のアドレスが指定されています。

```
_dmarc.hrtep.com  TXT  "v=DMARC1; p=none; fo=1; rua=mailto:mori@hrtep.com"
```

DMARC レポートは、ドメインを名乗る送信メールがどの IP から送られているかをドメイン所有者に通知する仕組みで、SES の本番運用後は AWS からの送信状況も該当アドレスにレポートが届く可能性があります。

### ご確認いただきたい事項

**`mori@hrtep.com` というアドレスが現存し、現在も受信可能な状態でしょうか？**

- 業務影響は通常ありません（DMARC レポートは運用監視用途）
- ただし当該アドレスが廃止されている場合、運用上の死角となります（送信不正監視レポートが受信できない状態）

### 想定される回答パターン

| パターン               | 必要な後続対応                                                                                                                                 |
| ---------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------- |
| (a) 現存・受信可能     | 何もしない。Phase 3 切替後もそのまま継続運用                                                                                                   |
| (b) 既に廃止／受信不能 | DMARC レコードの `rua=mailto:` 部分を別の管理用アドレス（貴社で運用継続している社内アドレス）に変更する DNS 更新を、安井様への次回依頼に含める |

### 回答欄

```
[ ] (a) mori@hrtep.com は現存し、受信可能
[ ] (b) mori@hrtep.com は既に廃止／受信不能 → 新たな受信先: ________________
[ ] (c) わからない／ご確認が必要 → leadx で調査を支援可能（Google Workspace 管理画面の閲覧権限が必要）
```

## ACTION 03 — お問合せフォームの社内通知メール送信先アドレス決定

### 背景

新サイトのお問合せフォーム（https://hrtep.com/contact）から送信があった際、貴社内の問合せ担当者宛に「お問合せ内容を社内に転送するメール」を 1 通送信いたします。本メールの宛先となるアドレスを貴社にてご決定いただきたく存じます。

このアドレスは：

- AWS Amplify Hosting の環境変数 `CONTACT_EMAIL_TO` に設定されます（コードにハードコードはしません）
- SES Production Access 申請文（弊社作成済）の本文中にも明記する必要があります（AWS への申請書類）
- 将来、別のアドレスへ変更する場合は環境変数の変更のみで切替可能です

### 候補（あくまで例示）

| 候補                    | 用途                                           |
| ----------------------- | ---------------------------------------------- |
| `info@hrtep.com`        | 一般的な代表問合せ受付アドレス                 |
| `contact@hrtep.com`     | 専用受付アドレス                               |
| `mori@hrtep.com`        | 既存運用アドレス（DMARC rua にも使われている） |
| `web-inquiry@hrtep.com` | Web 専用受付アドレス（新規作成）               |
| その他                  | （ご指定）                                     |

### 制約

- Google Workspace で受信可能なアドレスである必要があります（既存 MX 構成の延長線上）
- 個人アドレスではなく**業務アドレス**を推奨いたします（担当者異動時の引き継ぎを考慮）
- 1 アドレスのみ指定可能（複数宛先には現状の実装は対応しておりません。複数受信が必要な場合は、Google Workspace 側でグループ設定もしくは転送設定でご対応ください）

### 回答欄

```
社内通知メールの送信先アドレス: ____________________________________

副担当者への展開方法（任意）:
[ ] Google Workspace のグループアドレスを使う
[ ] Google Workspace の転送設定で複数アドレスに展開する
[ ] 単一アドレスのみで運用（副担当者への展開は別途検討）
```
