# HRtep株式会社 コーポレートサイトリニューアル：詳細設計

## 1. 修正内容の詳細化

### 1.1 プロジェクト概要

HRtep株式会社の既存コーポレートサイト（WordPress製・https://hrtep.com/）を、Next.js製の新サイトに全面リニューアルする。既存の外国人材紹介事業に加え、新たに展開するIT事業（DX支援・AI活用支援・システム開発・Webアプリ開発）を明確に訴求し、IT事業のポートフォリオも掲載する。

### 用語方針

| 領域         | SEO重視箇所（見出し・meta・タイ���ル） | ブランド表現箇所（理念・挨拶・About）     |
| ------------ | -------------------------------------- | ----------------------------------------- |
| 外国人材事業 | 「外国人材紹介」「外国人採用支援」     | 「外国籍人財」（HRtep独自のブランド表記） |
| AI関連       | 「AI活用支援」「AI導入支援」           | 同左                                      |
| Web開発      | 「Webアプリ開発」（省略形）            | 「Webアプリケーション開発」（正式名称）   |
| DX           | 「DX支援」                             | 同左                                      |
| システム開発 | 「システム開発」                       | 同左                                      |

**根拠**: 「AI業���効率化」はサービス名として業界でほぼ使われておらず、「AI活用支援」「AI導入支援」が標準。「外国籍人財」はSEOで「��国人材」に大きく劣るため、検索対象箇所では「外国人材」を使い、ブランド表現でのみ「人財」を使う。

### 1.2 何を・なぜ・どう変えるか

| 項目                 | Before（現行WordPress）                      | After（新Next.jsサイト）                             | 理由                                   |
| -------------------- | -------------------------------------------- | ---------------------------------------------------- | -------------------------------------- |
| **技術基盤**         | WordPress（PHP）                             | Next.js 16 + React 19 + TypeScript + Tailwind CSS v4 | 高速表示・型安全・保守性向上・開発効率 |
| **事業訴求**         | 外国人材事業が主、SI事業は1セクションのみ    | 外国人材事業とIT事業を対等に訴求                     | IT事業の追加が本リニューアルの主目的   |
| **ポートフォリオ**   | なし                                         | IT事業の実績一覧ページを新設                         | 顧客の発注判断材料の補強               |
| **サイト構造**       | 7ページ＋ブログ系コンテンツ                  | 10ページ（事業別詳細＋ポートフォリオ）               | 事業ごとに十分な情報量を確保           |
| **ニュース**         | 2022年で更新停止（3件）                      | 静的データ管理、将来CMS化対応                        | 最新情報の発信基盤を整備               |
| **デザイン**         | WordPress既製テーマ                          | オリジナルデザイン（Tailwind CSS）                   | HRtep独自のブランドアイデンティティ    |
| **レスポンシブ**     | テーマ依存                                   | モバイルファーストで全ページ対応                     | モバイルユーザーの利便性確保           |
| **SEO**              | WordPress SEOプラグイン依存                  | Next.js Metadata API + 構造化データ                  | 検索エンジン最適化の本質的な実装       |
| **お問い合わせ**     | WordPress Contact Form + reCAPTCHA           | Next.js API Route + メール送信サービス               | サーバーレス対応・保守性向上           |
| **不要ページの廃止** | お役立ち情報・必ず読んでほしい記事・無料登録 | 廃止（コーポレートサイトに不要）                     | 事業訴求に集中、情報の希薄化を防止     |

#### お問い合わせページ（`/contact`）

- フォームフィールド：会社名、氏名、メールアドレス、電話番号、お問い合わせ種別（外国人材紹介/DX支援/AI活用支援/システム開発/Webアプリ開発/運用・保守/IT教育・研修/その他）、お問い合わせ内容
- バリデーション（クライアント: react-hook-form + zod で即時フィードバック / サーバー: Server Action 内で zod による二重検証）
- 送信後サンクスメッセージ表示
- バックエンド: **Server Action**（`app/contact/actions.ts`）+ Resend（メール送信サービス）。API Route は使用しない（CLAUDE.md「API Routeは外部連携等の必要時のみ」に準拠）
- UIパターン: shadcn/ui `Form`（react-hook-formラッパー）+ `FormField` / `FormItem` / `FormLabel` / `FormControl` / `FormMessage` で構成。`handleSubmit` コールバックから Server Action を呼び出す
- **注**: react-hook-form の `handleSubmit` はクライアントJS必須のため、Progressive Enhancement（JS無効環境での送信）は不成立。コーポレートサイトの利用環境（現代ブラウザ）では実害なし

### 1.8 コンテンツ変更容易性の設計原則

たたき台を見ながらコンテンツ（テキスト・画像・フォント・色）を臨機応変に差し替えるため、以下の分離原則を徹底する。

| 変更対象               | 変更箇所                                                               | 変更影響範囲                           |
| ---------------------- | ---------------------------------------------------------------------- | -------------------------------------- |
| **テキスト内容**       | `data/*.ts` / `lib/constants.ts`                                       | データファイルのみ。コンポーネント不変 |
| **画像**               | `public/images/` のファイル差し替え + `data/*.ts` のパス参照           | 画像ファイルのみ。コンポーネント不変   |
| **ブランドカラー**     | `app/globals.css` の `@theme inline` カスタムプロパティ                | 1ファイルの変数値変更で全サイトに反映  |
| **フォント**           | `app/layout.tsx` の `next/font` 定義 + `app/globals.css` のfont-family | 2ファイルの変更で全サイトに反映        |
| **セクション順序**     | 各 `page.tsx` のコンポーネント配置順                                   | 該当ページファイルのみ                 |
| **ナビゲーション項目** | `data/navigation.ts`                                                   | データファイルのみ。Header/Footer不変  |

#### フォント戦略

| 用途       | フォント                           | 設定                         |
| ---------- | ---------------------------------- | ---------------------------- |
| 日本語本文 | Noto Sans JP（`next/font/google`） | weight: 400, 500, 700        |
| 英語・数字 | Geist（現行設定を維持）            | weight: 400, 500, 600, 700   |
| 見出し英字 | Geist                              | 大きめサイズで使用           |
| コード     | Geist Mono（現行設定を維持）       | 使用箇所なし（将来用に維持） |

`next/font/google` の `Noto_Sans_JP` を追加し、CSS変数 `--font-noto` として登録。`body` に `font-family: var(--font-noto), var(--font-geist-sans), sans-serif` を適用。

### 1.9 多言語対応（i18n）戦略

| フェーズ                   | 対応範囲      | 方針                                                                                                                      |
| -------------------------- | ------------- | ------------------------------------------------------------------------------------------------------------------------- |
| **たたき台（現フェーズ）** | 日本語のみ    | コンテンツは `data/` に分離済みのため、将来の多言語化を阻害しない。ヘッダーに言語切替UIは設けない                         |
| **将来（必要時）**         | 日本語 + 英語 | Next.js App Router の `[locale]` セグメント + middleware でルーティング。`data/` ファイルを `data/ja/`, `data/en/` に分割 |

**日本語のみとする根拠**: コーポレートサイトの主要ターゲットは日本企業（B2B）。外国人材自身はHRtepプラットフォーム側を利用するため、コーポレートサイトの英語化は初期段階では不要。ただし、外国人材事業を展開する企業として、将来の英語対応を見据え、テキストのハードコーディングは行わない。

### 1.10 アクセス解析・Cookie同意

| 項目                 | 方針                                                                                           |

| `app/layout.tsx`  | `<html lang="ja">` を明示設定（WCAG 3.1.1）。Noto Sans JP を追加（`next/font/google`）、Geistフォントと併用。`<Header>` と `<Footer>` を children の前後に配置。`metadata` を HRtep用に全面書き換え（title template、description、OGP、Organization JSON-LD）。`viewport` 設定追加 |

#### 新規ファイルの主要方針

**レイアウト系（`components/layout/`）**

- `Header.tsx`: Server Component。ロゴ（next/image）+ Navigation + MobileMenu。スクロール時の背景色変化はClient Componentで分離
- `Footer.tsx`: Server Component。ロゴ、住所・TEL、ナビリンク、プライバシーポリシーリンク、コピーライト
- `MobileMenu.tsx`: Client Component（`use client`）。shadcn/ui `Sheet` をベースに実装。ハンバーガーアイコンのトグル、スライドインメニュー、フォーカストラップ・body スクロールロックはRadix UIが自動処理

**画像（`next/image` 利用方針）**

- 全画像は `next/image`（`<Image>` コンポーネント）を使用。`<img>` タグの直接使用は禁止
- 自動 WebP/AVIF 変換、レスポンシブ `srcSet` 生成、lazy loading がデフォルトで有効
- Hero画像など Above the fold の画像には `priority` 属性を付与（lazy loading 無効化）
- 装飾画像は `alt=""` + `aria-hidden="true"`、意味のある画像は適切な代替テキストを設定（WCAG 1.1.1）

**UI基盤（`components/ui/`）**

- shadcn/ui のコンポーネント（Button, Card, Input, Textarea, Select, Label, Sheet, NavigationMenu, Separator）が `components/ui/` に自動配置される
- プロジェクト固有のUIコンポーネント（以下）も同ディレクトリに配置：
  - `SectionHeading.tsx`: 英語タイトル（大）+ 日本語サブタイトル（小）のパターンを共通化（現行サイトのデザインパターンを踏襲）
  - `Container.tsx`, `Breadcrumb.tsx`, `PageHero.tsx`, `CTABanner.tsx`

**ページ固有（`components/home/`, `components/about/` 等）**

- 各セクションを独立コンポーネントとして分離し、ページファイルの可読性を確保
- データは `data/` ディレクトリの定義を import して使用（ハードコーディング回避）

**データ層（`data/`）**

- TypeScript ファイルで型付きデータを定義。将来的に CMS/API に差し替え可能な構造
- `data/services.ts`: サービス名、概要、アイコン名、詳細リンク等
- `data/portfolio.ts`: プロジェクト名、カテゴリ、概要、サムネイルパス、技術スタック等
- `data/news.ts`: slug、タイトル、日付、カテゴリ、本文（MDXまたはHTML文字列）

**Server Action（`app/contact/actions.ts`）**

- `'use server'` ディレクティブ付きの非同期関数。zod でバリデーション → Resend でメール送信 → 結果オブジェクト返却
- react-hook-form の `action` prop と連携。Progressive Enhancement 対応
- Rate limiting は middleware で実装
- 環境変数: `RESEND_API_KEY`, `CONTACT_EMAIL_TO`
